Purpose: This is the mathematical foundation for all sampling improvements in Phase 19. The IS estimator generalizes the current N_det/N_total histogram to handle non-uniform proposal distributions. Once in place, any change to the injection campaign sampling strategy is automatically handled by computing appropriate weights -- no further estimator changes needed.

</objective>

  <files>master_thesis_code/bayesian_inference/simulation_detection_probability.py, analysis/importance_sampling.py</files>
  <action>
    Create a standalone IS utility module `analysis/importance_sampling.py` and extend `_build_grid_2d` with weight support.

    **Step 1: Create `analysis/importance_sampling.py`** with pure functions:

    1. `weighted_histogram_estimator(detected_weights, total_weights) -> float`:
       - Computes P_hat = sum(detected_weights) / sum(total_weights)
       - If sum(total_weights) == 0, return 0.0 (conservative, same as current behavior)
       - This is the self-normalized IS estimator from Tiwari (2018) Eq. 5-8
       - When all weights are 1.0, this is exactly N_det / N_total

    2. `kish_n_eff(weights: NDArray) -> float`:
       - Computes N_eff = (sum(w))^2 / sum(w^2) per Kish (1965)
       - If len(weights) == 0 or sum(w) == 0, return 0.0
       - When all weights are equal, N_eff = len(weights) exactly

    3. `is_weighted_wilson_ci(detected_weights, total_weights, confidence_level=0.9545) -> tuple[float, float]`:
       - Wilson CI adapted for effective sample size: use N_eff in place of n in the Wilson formula
       - This gives wider CIs when weights are unequal (correctly reflecting reduced information)
       - Reference: Brown, Cai, DasGupta (2001) with N_eff substitution per Owen (2013)

    All functions must have full type annotations, NumPy-style docstrings, and reference comments.

    **Step 2: Extend `_build_grid_2d` in `SimulationDetectionProbability`**:

    Modify the method signature to accept an optional `weights` parameter:
    ```
    def _build_grid_2d(self, df, snr_threshold, h_val=None, weights=None):
    ```

    Implementation:
    1. If `weights is None`, set `weights = np.ones(len(df))` (backward compat)
    2. Use `np.digitize` to assign each injection to a (d_L, M) bin (same bin edges as current)
    3. For each bin, accumulate:
       - `total_weights[i,j] = sum(w_k for k in bin(i,j))`
       - `detected_weights[i,j] = sum(w_k for k in bin(i,j) if SNR_k >= threshold)`
    4. Compute `p_det_grid[i,j] = detected_weights[i,j] / total_weights[i,j]` (0 if total=0)
    5. Compute `n_eff[i,j] = kish_n_eff(weights_in_bin[i,j])`
    6. Store `n_eff` in quality_flags alongside existing n_total, n_detected, reliable

    CRITICAL: The bin edges (dl_edges, M_edges) and centers (dl_centers, M_centers) must remain IDENTICAL to the current implementation. Only the counting logic changes from integer counts to weighted sums.

    Use `np.add.at` or manual loop over bins for weight accumulation. Do NOT use `np.histogram2d` with `weights=` parameter for the detected histogram because it would require pre-filtering -- instead, accumulate both total and detected weights in a single pass through the data using digitized bin indices.

    The existing quality_flags dict gains one new key: `"n_eff"` (float64 array same shape as n_total).
    The existing `"reliable"` mask definition stays as `n_total >= 10` (integer counts, not weighted).
  </action>
