  <action>
    Create `analysis/validation.py` with the following functions, all operating on Phase 18 injection data via `analysis/grid_quality.py` and `analysis/importance_sampling.py`:

    1. `wilson_ci_overlap_test(grid_standard: GridResult, grid_is: GridResult, min_n: int = 10) -> dict`:
       - For each bin where BOTH grids have n_total >= min_n:
         - Get Wilson CI from standard estimator (GridResult.ci_lower, ci_upper)
         - Get Wilson CI from IS estimator (using `is_weighted_wilson_ci` with w=1 weights passed through)
         - Two CIs overlap if ci_lower_A <= ci_upper_B AND ci_lower_B <= ci_upper_A
         - Record bin index, P_det values, CI bounds, and overlap boolean
       - Return dict with: n_tested, n_overlap, n_non_overlap, per_bin_results list, eligible_mask

    2. `bh_fdr_correction(non_overlap_flags: list[bool], q: float = 0.05) -> dict`:
       - Input: boolean flag per bin (True = non-overlap = "discovery candidate")
       - Convert to p-values: overlapping -> p=1.0, non-overlapping -> p=0.0 (conservative: treat non-overlap as p=0)
       - More precisely: for w=1 case where IS and standard are mathematically identical, ALL bins overlap, so p-values are moot.
       - For the general case (future non-uniform weights): use the CI overlap as the test; the BH correction operates on the fraction of non-overlapping bins.
       - Implement BH procedure: sort p-values ascending, reject p_(i) if p_(i) <= (i/m)*q for largest such i
       - Return dict with: n_discoveries, discovery_indices, q_value, n_tests

    3. `monotonicity_check(grid: GridResult) -> dict`:
       - For each M-column (j = 0, ..., M_bins-1):
         - Extract P_det vs d_L_center for bins with n_total >= 10
         - Use `sklearn.isotonic.IsotonicRegression(increasing=False)` to fit monotone non-increasing function
         - Compute residuals: P_det_observed - P_det_isotonic
         - Flag violations where |residual| > 2 * CI_half_width AND n_total >= 10
       - Return dict with: n_columns_tested, violations list (each with M_col_index, d_L_bin_index, residual, ci_hw, n_total), n_significant_violations

    4. `boundary_condition_check(grid: GridResult) -> dict`:
       - Low-d_L, high-M corner: check P_det in bin (0, M_bins-1) or nearest non-empty bin
       - High-d_L, low-M corner: check P_det in bin (dl_bins-1, 0) or nearest non-empty bin
       - Also check: P_det < 0.05 in the highest-d_L row (all M values)
       - Return dict with: low_corner_pdet, high_corner_pdet, high_dl_row_max_pdet, pass_low (P_det > 0.5 or n_total=0), pass_high (P_det < 0.05 or n_det=0)

    5. `run_validation(data_dir: str | Path, h_values: list[float] | None = None) -> dict`:
       - Main entry point. For each h-value:
         a. Load injection data with `load_injection_data(data_dir, h_val)`
         b. Build 15x10 grid with `build_grid_with_ci(...)` for standard estimator
         c. Build IS-weighted grid with w=1 (using `weighted_histogram_estimator` from importance_sampling module, passed through `_build_grid_2d` or equivalent standalone computation)
         d. Run `wilson_ci_overlap_test`
         e. Run `monotonicity_check`
         f. Run `boundary_condition_check`
         g. Run `farr_criterion_check` from importance_sampling module
       - Pool all per-bin non-overlap flags across h-values, run `bh_fdr_correction`
       - Return nested dict with per-h results and global BH result

    CRITICAL: The w=1 IS estimator MUST produce P_det identical to the standard estimator (max |diff| = 0.0 was already verified in Phase 19-01). The Wilson CI overlap test is therefore expected to show 100% overlap. The framework must be designed so that when real non-uniform weight data exists, the same functions produce meaningful comparisons.
