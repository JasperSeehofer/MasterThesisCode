  <name>Task 1: Implement grid-vs-MC comparison</name>
  <files>analysis/validation.py</files>
  <action>
    Add the following functions to `analysis/validation.py` (extending the module created by Plan 20-01, or creating a standalone section if Plan 20-02 executes first):

    1. `direct_mc_alpha(n_detected: int, n_total: int) -> float`:
       - Compute alpha(h) = N_det / N_total
       - This is Eq. (8) of Mandel, Farr & Gair (2019): the fraction of injected events that are detected
       - Return float in [0, 1]
       - Reference comment: # Eq. (8) of Mandel, Farr & Gair (2019), arXiv:1809.02063

    2. `grid_integrated_alpha(grid: GridResult) -> float`:
       - Compute the grid-based approximation to alpha(h):
         alpha_grid = sum_B [ P_det(B) * N_total(B) ] / N_total_global
       - This weights each bin's P_det by its occupancy fraction, reconstructing the global detection fraction from the binned estimates
       - Mathematically: alpha_grid = sum_B (N_det(B)/N_total(B)) * (N_total(B)/N_total) = sum_B N_det(B) / N_total = alpha_MC exactly
       - IMPORTANT: For the standard estimator P_det = N_det/N_total, this identity holds EXACTLY. Any difference indicates a bug in the grid construction (e.g., events falling outside bin edges).
       - Reference comment: # Grid-integrated selection integral, cf. Mandel, Farr & Gair (2019)

    3. `grid_vs_mc_comparison(data_dir: str | Path, h_values: list[float] | None = None) -> dict`:
       - For each h-value:
         a. Load injection data with `load_injection_data(data_dir, h_val)`
         b. Compute alpha_MC = N_det / N_total from raw counts
         c. Build 15x10 grid with `build_grid_with_ci(...)`
         d. Compute alpha_grid from grid
         e. Compute |alpha_grid - alpha_MC| (absolute difference)
         f. Compute MC uncertainty: sigma_MC = sqrt(alpha*(1-alpha)/N)
         g. Compute relative difference: |alpha_grid - alpha_MC| / sigma_MC (in units of sigma)
       - Return dict with per-h results and global summary

    MATHEMATICAL NOTE: For the standard (unweighted) estimator, alpha_grid = alpha_MC is an algebraic identity -- any nonzero difference means events are being lost or double-counted in the binning. The interesting comparison arises when IS weights are applied (future work) or when the grid is used for interpolation at non-bin-center points.

    Even though the identity holds exactly, computing both independently provides a round-trip consistency check on the grid construction pipeline (load_injection_data -> build_grid_with_ci -> sum). This catches silent bugs like bin-edge events being dropped.

    Use same typing conventions as Task 1. Add reference comments above formulas.
