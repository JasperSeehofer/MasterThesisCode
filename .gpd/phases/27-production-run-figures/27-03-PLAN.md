<task type="auto">
  <name>Task 1: Generate posterior convergence figure</name>
  <files>master_thesis_code/plotting/paper_figures.py, paper/figures/posterior_convergence.pdf</files>
  <action>
    Add a function `plot_posterior_convergence()` to `master_thesis_code/plotting/paper_figures.py` (created in Plan 02).

    Algorithm:
    1. Load all per-event posteriors from `cluster_results/eval_corrected_full/posteriors/` for the WITHOUT-BH-mass variant ONLY.
       NOTE: The per-event files use a 23-point h-grid (finer than the 15-point combined posterior grid). Use all 23 h-values for convergence analysis -- this gives better CI resolution than the combined grid.
    2. For each h-value file, build an array of shape (n_events,) with the single posterior value per event
    3. Construct the full event_posteriors matrix: shape (n_events, n_h_values) where entry [i, j] = posterior value for event i at h-value j. Expect n_h_values = 23.
    4. Generate N_subsets = 50 random subsets at each subset size N_sub = [10, 20, 50, 100, 150, 200, 300, 400, 500]
    5. For each subset:
       - Combine the N_sub event posteriors in log-space: log_combined = sum of log(p_i(h)) over events in subset
       - Normalize: log_combined -= max(log_combined); combined = exp(log_combined)
       - Compute 68% CI width using CDF interpolation (same method as Plan 02 Task 1, but on the 23-point grid which provides better resolution)
    6. Plot: x = N_sub (log scale), y = CI_width (log scale)
       - Points: median CI width across 50 subsets
       - Error bars: 16th-84th percentile range
       - Dashed reference line: N^{-1/2} scaling anchored to the N=500 median

    OMIT the with-BH-mass convergence curve entirely. Reason: the with-BH-mass combined posterior is effectively a delta function (only 1 bin with nonzero mass at h=0.68). Computing CI width vs N for a delta-function posterior is scientifically meaningless -- it would show the CI collapsing to grid-limited (< grid_spacing) at very few events and staying there. Additionally, loading the with-BH-mass per-event files (~610 MB each, 8 files = ~5 GB) is prohibitively expensive for a meaningless result. Note the omission and reasoning in the SUMMARY.

    Use `get_figure(width='single')`, `apply_style()`, `save_figure()`.
    x-label: "Number of events $N_\\mathrm{det}$"
    y-label: "$1\\sigma$ width of $h$ posterior"
    Legend: "Without $M_z$"
    Save to `paper/figures/posterior_convergence.pdf`.

    Record the N_subsets and subset sizes used in a comment for filling \pending{} markers.
  </action>
  <verify>
    1. Figure shows decreasing CI width with increasing N (expected statistical behavior)
    2. Slope on log-log is approximately -0.5 (consistent with N^{-1/2})
    3. Error bars decrease with N (more subsets converge to similar CI)
    4. N^{-1/2} reference line is visible
    5. Figure is valid PDF, publication quality
  </verify>
  <done>Convergence figure generated showing N^{-1/2} scaling of posterior width for without-BH-mass variant (using 23-point per-event grid). With-BH-mass omitted due to delta-function posterior. N_subsets and subset sizes recorded for paper markers.</done>
