**Section 1: Bayesian Framework**

Start from Bayes' theorem for H0 given N_det GW detections and a galaxy catalog:

  p(H0 | {d_GW}, catalog) proportional to p(H0) * product_{i=1}^{N_det} p(d_GW^i | H0, catalog)

For each detection, the single-event likelihood marginalizes over possible host galaxies:

  p(d_GW | H0, catalog) = sum_{j in galaxies} w_j * integral dz  [numerator terms] / [selection denominator]

Define every symbol: d_GW = GW data stream, H0 = Hubble constant, z = true redshift of host, w_j = galaxy weight (proportional to merger rate in that galaxy).

**Section 2: Term-by-term d_L-only likelihood**

Write the single-host-galaxy numerator integrand as a product of:

1. **GW likelihood** p(d_GW | phi, theta, d_L(z, H0)): the probability of the observed GW data given source sky position and luminosity distance. Under Fisher matrix approximation, this is a multivariate Gaussian in (phi, theta, d_L/d_L_measured) centered at the ML estimates.

2. **Galaxy redshift prior** p_gal(z): the probability of the galaxy being at redshift z, modeled as a Gaussian N(z; z_gal, sigma_z) from the galaxy catalog.

3. **Detection probability** p_det(d_L, phi, theta): the probability that a source at (d_L, phi, theta) would be detected (SNR >= threshold). This appears in the selection correction denominator.

4. **Volume element**: State explicitly what the integration measure dz brings. The luminosity distance d_L(z, H0) depends on H0 through the distance-redshift relation. The Jacobian dd_L/dz is absorbed into the GW likelihood parameterized as d_L_frac = d_L(z,H0)/d_L_measured.

5. **Sky localization weight**: Resolve WHERE the (phi, theta) dependence enters. In the code, the GW likelihood Gaussian is 3D in (phi, theta, d_L_frac). This means sky localization is INSIDE the GW likelihood, not a separate factor. Derive this from the Fisher matrix structure: the full GW posterior is over all parameters including sky position, so the 3D Gaussian already encodes the angular information.

Write out the denominator integrand: p_det(d_L, phi, theta) * p_gal(z). Explain why: the denominator accounts for the probability that a source from this galaxy WOULD be detected, regardless of where it is. The GW likelihood does not appear in the denominator because it integrates to 1 over the data.

Explicitly state the structure:
- Numerator: p_det * p_GW(phi, theta, d_L_frac) * p_gal(z)
- Denominator: integral dz [ p_det * p_gal(z) ]

Note: The notation d_L_frac = d_L(z, H0) / d_L_measured is used because the Fisher matrix covariance is expressed in fractional units.

**Section 3: Connection to code**

Map each term to the code variables:
