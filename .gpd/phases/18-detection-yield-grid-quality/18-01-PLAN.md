  <name>Task 1: Compute detection yield per h-value and waste breakdown</name>
  <files>analysis/injection_yield.py</files>
  <action>
    Create a standalone analysis script that loads all 262 injection CSVs from simulations/injections/ and computes:

    1. Per-h-value detection yield:
       - Group CSV files by h-value (7 groups: h=0.60, 0.65, 0.70, 0.73, 0.80, 0.85, 0.90)
       - Load all CSVs per group using pd.read_csv + pd.concat
       - Count N_total = len(df) per h (total successful waveform evaluations in CSV)
       - Count N_det = sum(SNR >= 15) per h
       - Count N_sub_threshold = N_total - N_det per h
       - Compute f_det = N_det / N_total to at least 3 significant figures
       - Store results in a summary DataFrame

    2. CSV-only waste decomposition (exact from data):
       - Per h: fraction_detected = N_det / N_total
       - Per h: fraction_sub_threshold = N_sub_threshold / N_total
       - Verify: fraction_detected + fraction_sub_threshold = 1.0 exactly

    3. Estimated 3-way waste decomposition (including failures):
       - SLURM logs are NOT available locally. CSV records only successful waveform evaluations.
       - Estimate total attempted = N_total / (1 - failure_rate)
       - Report at TWO assumed failure rates: 30% and 50% (bracketing Phase 17 estimate)
       - Per h and per scenario:
         * fraction_failed = failure_rate (assumed)
         * fraction_sub_threshold = N_sub_threshold / N_attempted
         * fraction_detected = N_det / N_attempted
         * Verify fractions sum to 1.0

    4. z > 0.5 cutoff validation:
       - Per h: count events with z > 0.5 AND SNR >= 15
       - Expected: zero for all 7 h-values (Phase 17 confirmed z_cut=0.5 safe)
       - Also report: max detected z per h-value (for reference)

    5. Farr (2019) criterion check:
       - Per h: check if N_total > 4 * N_det (N_eff > 4*N_det for uniform weights)
       - With uniform injection weights, N_eff = N_total, so the criterion simplifies

    Use SNR_THRESHOLD = 15 from constants.py. Use the filename pattern injection_h_{h}p{hh}_task_{n}.csv to extract h-values.

    Structure the script with clear functions: load_injection_data(), compute_yield(), compute_waste(), validate_zcutoff(), check_farr_criterion(). Include a __main__ block that runs all analyses and prints a summary table.

    IMPORTANT: The CSV columns are z, M, phiS, qS, SNR, h_inj, luminosity_distance (per Phase 17 audit). Use 'z' column for z-cutoff check, 'SNR' column for detection threshold.
  </action>
  <verify>
    1. N_sub_threshold + N_det = N_total exactly for each h (integer accounting)
    2. f_det for all 7 h-values reported to 3+ significant figures
    3. f_det approximately monotonically increasing with h (higher h -> closer sources -> more detections)
    4. Cross-check: total events ~165,000 across all h (matching Phase 17 report)
    5. Estimated waste fractions sum to 1.0 for both 30% and 50% failure scenarios
    6. Farr criterion: N_total / N_det >> 4 for all h (expected: N_total/N_det ~ 250-750)
  </verify>
