**Dev partition tip (per D-07):**
- Callout box:
> **Tip:** Test with a small run first before submitting a large campaign:
> ```bash
> bash cluster/submit_pipeline.sh --tasks 2 --steps 10 --seed 42
> ```
> bwUniCluster has a `dev_gpu_h100` partition with shorter queue times for testing. To use it, edit `simulate.sbatch` and change `--partition=gpu_h100` to `--partition=dev_gpu_h100`.

`## Monitoring`
- `squeue -u $USER` -- check queue status
- `sacct -j JOB_ID --format=JobID,State,Elapsed,MaxRSS,ExitCode` -- detailed job info
- `tail -f $WORKSPACE/run_*/logs/simulate_*.out` -- follow live output
- The `submit_pipeline.sh` output prints a ready-to-use `sacct` command with all three job IDs.

`## Retrieving Results`
- From local machine:
  ```bash
  scp -r bwunicluster:$(ws_find emri)/run_20260327_seed42/ ./results/
  ```
- Or use `rsync` for incremental transfer:
  ```bash
  rsync -avz bwunicluster:PATH/run_20260327_seed42/ ./results/
  ```

Workspace expiration warning (per D-13):
> **Warning:** Workspace `emri` expires after 60 days. Copy results to persistent storage before expiration. Check remaining time: `ws_list`. Extend: `ws_extend emri 60`.

`## Troubleshooting` (per D-12, D-14)
- **OOM kills:** `sacct -j JOB_ID --format=JobID,State,MaxRSS` shows memory usage. State will be `OUT_OF_MEMORY`. Reduce `--simulation_steps` or request more memory.
- **Timeout failures:** State `TIMEOUT`. Increase `--time` in `simulate.sbatch`. Default is 2 hours.
- **CUDA errors:** Check `cluster/logs/simulate_*.err` for `CUDA error` or `cupy.cuda.runtime.CUDARuntimeError`. Ensure `module load devel/cuda/12.8` succeeded.
- **Python tracebacks:** `grep -l "Traceback" $RUN_DIR/logs/*.err` to find which tasks failed.
- **Resubmitting failed tasks:**
  ```bash
  bash cluster/resubmit_failed.sh JOB_ID $RUN_DIR BASE_SEED SIM_STEPS
  ```
  This queries sacct for FAILED/TIMEOUT/NODE_FAIL/OUT_OF_MEMORY tasks, cleans up partial output, and resubmits only those array indices. After completion, manually resubmit merge+evaluate (the script prints the exact command).

**Log file locations (per D-14):**
- SLURM stdout: `$RUN_DIR/logs/simulate_JOBID_TASKID.out`
- SLURM stderr: `$RUN_DIR/logs/simulate_JOBID_TASKID.err`
- Key grep patterns:
  ```bash
  grep -r "ERROR\|Traceback\|CUDA error\|OOM\|Killed" $RUN_DIR/logs/
  ```

`## Script Reference` (per D-10)
Table listing each script with a one-line description:

| Script | Purpose |
|--------|---------|
| `modules.sh` | Loads environment modules; exports `$WORKSPACE`, `$PROJECT_ROOT`, `$VENV_PATH` |
| `setup.sh` | First-time setup: installs uv, allocates workspace, creates venv |
| `simulate.sbatch` | GPU array job -- one EMRI simulation per task |
| `merge.sbatch` | CPU job -- merges per-task CSVs, prepares detections |
| `evaluate.sbatch` | CPU job -- Bayesian inference for H0 posterior (16 CPUs) |
| `submit_pipeline.sh` | Pipeline orchestrator -- chains simulate -> merge -> evaluate |
| `resubmit_failed.sh` | Resubmits only failed array tasks |
| `vpn.sh` | University VPN connection via openconnect |

**Seed strategy** subsection:
- Per-task seed = `BASE_SEED + SLURM_ARRAY_TASK_ID`
- This ensures: (1) each task gets a unique random state, (2) results are reproducible given the same base seed, (3) resubmitted tasks produce identical results.
- Seeds and all SLURM metadata are recorded in `run_metadata_N.json`.
