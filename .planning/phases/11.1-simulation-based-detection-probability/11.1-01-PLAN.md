Note: The injection campaign does NOT need the galaxy_catalog (per D-04 and RESEARCH.md -- events are drawn from the population model directly, not from the GLADE catalog).

**`injection_campaign()` function** -- simplified version of `data_simulation()` that:

1. Takes parameters: `simulation_steps: int, cosmological_model: Model1CrossCheck, h_value: float, simulation_index: int, rng: np.random.Generator, *, use_gpu: bool = False`
2. Imports `ParameterEstimation`, `WaveGeneratorType`, `MemoryManagement` (same conditional imports as data_simulation)
3. Imports `dist` from `physical_relations`
4. Imports `INJECTION_CSV_PATH` from `constants`
5. Creates `ParameterEstimation` with `cosmological_model.parameter_space` and `use_gpu`
6. Sets up SIGALRM handler for 90s timeout (same pattern as data_simulation)
7. Creates output directory: `os.makedirs(os.path.dirname(csv_path), exist_ok=True)` where `csv_path` is `INJECTION_CSV_PATH` with `{h_label}` replaced by `str(round(h_value, 3)).replace('.', '_')` and `$index` replaced by `str(simulation_index)`
8. Runs a while loop counting SUCCESSFUL SNR computations up to `simulation_steps`:
   a. Sample events from `cosmological_model.sample_emri_events(200)` (same batch pattern as data_simulation)
   b. `parameter_estimation.parameter_space.randomize_parameters(rng=rng)`
   c. Set M from sample: `parameter_estimation.parameter_space.M.value = sample.M`
   d. Set sky angles from population model: use values from `randomize_parameters()` (phiS and qS are already randomized uniformly by `randomize_parameters()`)
   e. **CRITICAL per D-04:** Set luminosity distance with candidate h: `parameter_estimation.parameter_space.luminosity_distance.value = dist(sample.redshift, h=h_value)` -- do NOT use `set_host_galaxy_parameters()` which hardcodes h=0.73
   f. Try `snr = parameter_estimation.compute_signal_to_noise_ratio()` with same exception handling as data_simulation (Warning, ParameterOutOfBoundsError, RuntimeError, ValueError/EllipticK, ZeroDivisionError, TimeoutError)
   g. On success: append row to results list with columns: `z, M, phiS, qS, SNR, h_inj, luminosity_distance` (per D-01, D-03 -- store raw SNR, do NOT threshold)
   h. Increment counter on successful SNR computation (per RESEARCH Pitfall 5 -- count successes, not attempts)
   i. Free GPU memory periodically (same `memory_management.free_gpu_memory()` pattern)
9. After loop: write results to CSV using pandas DataFrame with columns `["z", "M", "phiS", "qS", "SNR", "h_inj", "luminosity_distance"]`
10. Log summary: total events stored, path to CSV

Key differences from `data_simulation()`:
- No galaxy catalog lookup (no `GalaxyCatalogueHandler`)
- No quick SNR pre-check (compute full SNR directly per D-07)
- No Fisher matrix / CRB computation (per D-07)
- No `save_not_detected()` -- store ALL events regardless of SNR (per D-03)
- No `save_cramer_rao_bound()` -- only store injection CSV
- Uses `dist(z, h=h_value)` instead of `set_host_galaxy_parameters()` (per D-04)
- No callbacks (no plotting needed for injection campaign)
