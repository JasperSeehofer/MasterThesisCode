# Phase 11.1: Simulation-Based Detection Probability - Context

**Gathered:** 2026-03-30
**Status:** Ready for planning

<domain>
## Phase Boundary

Replace the KDE-based `DetectionProbability` class with a simulation-based detection probability built from a dedicated SNR-only injection campaign. For each candidate h value, draw synthetic EMRI events from the population model, compute SNR, and store raw results. P_det is computed at evaluation time by applying an SNR threshold and binning — mirroring the real LISA data analysis procedure (injection campaign methodology, cf. Gray et al. 2020, LVK dark siren analyses).

This phase covers: injection campaign code + SLURM job, new `SimulationDetectionProbability` class, Pipeline B integration, and validation on existing detections.

</domain>

<decisions>
## Implementation Decisions

### P_det Grid Design
- **D-01:** Build full 5D injection data: store (z, M, phi, theta, SNR, h_inj) per injected event. P_det grid can be constructed at any dimensionality from this raw data.
- **D-02:** For evaluation, start with full P_det(z, M, phi, theta | h). If too noisy, project down to P_det(z, M | h) by marginalizing over sky angles (physically motivated since angles are uniformly distributed).
- **D-03:** Store raw SNR values — do NOT pre-apply a detection threshold. This allows rebuilding P_det for different SNR thresholds without re-running the simulation.

### h-Dependence (Injection Campaign Methodology)
- **D-04:** P_det depends on h through the population model: for each candidate h, draw events at redshifts z, compute d_L(z, h), generate waveform, compute SNR. This mirrors real LISA data analysis where P_det is built from simulated injections, not from observed detections.
- **D-05:** Run separate injection campaigns per h value. Start with 5-7 h values: {0.60, 0.65, 0.70, 0.73, 0.80, 0.85, 0.90}. Can add more h points later without re-running existing ones.
- **D-06:** Interpolate P_det between h grid points at evaluation time.

### Campaign Scale & Strategy
- **D-07:** SNR-only dedicated campaign — skip Fisher matrix / CRB computation entirely. Only compute waveform + inner product for SNR. This is orders of magnitude faster per event.
- **D-08:** ~10,000 injection events per h value. With 7 h values: ~70,000 total events.
- **D-09:** Incremental accumulation: design the storage format so new simulation runs append to the existing injection CSV. Running the campaign again adds events for better P_det approximation without discarding prior data.

### Integration into Pipeline B
- **D-10:** Write a new `SimulationDetectionProbability` class that loads injection CSV(s), applies SNR threshold at evaluation time, and builds interpolation grid(s). Replace `DetectionProbability` in `BayesianStatistics`.
- **D-11:** Delete the old KDE-based `DetectionProbability` class (`detection_probability.py`) entirely.
- **D-12:** Remove the `_DEBUG_DISABLE_DETECTION_PROBABILITY` flag and all associated conditional logic in `bayesian_statistics.py`.
- **D-13:** Remove usage of `undetected_events.csv` in Pipeline B — the injection campaign replaces it.

### Validation
- **D-14:** Validation uses existing Phase 11 detection catalog (22 events) plus additional detections from ongoing cluster runs. Compare H0 posterior against the P_det=1 baseline (peaks at h=0.678 / h=0.600).
- **D-15:** Success criterion: formal statistical test — h=0.73 falls within the 90% credible interval of the combined posterior. This machinery is needed for the real analysis anyway.

### Claude's Discretion
- SLURM job structure for the injection campaign (array jobs, tasks per h value)
- Grid binning resolution and interpolation method (linear, cubic, etc.)
- Exact file format for injection CSV (column names, ordering)
- How to structure the new class (constructor API, method signatures)
- Whether to add a CLI flag for injection-only mode or a separate entry point

### Deferred Ideas
- Adding more h grid points for finer P_det resolution (run more injections later)
- Using injection data to calibrate the d_L error threshold (currently 10%)

</decisions>

## Specific Ideas

- Mirror real LISA data analysis: injection campaign methodology, not KDE on observed events
- Raw SNR storage enables rebuilding P_det at different thresholds without re-simulating
- Incremental design: "just keep running and adding events" for progressively better P_det
- 5-7 h grid points initially, expandable later without discarding existing data

</specifics>
