## P_det Grid Design

### Grid Dimensions

| Option | Description | Selected |
|--------|-------------|----------|
| P_det(z, h) only | 2D grid marginalized over sky angles and mass. Simplest. | |
| P_det(d_L, h) then convert | Grid in (d_L, h) space with z↔d_L conversion. | |
| P_det(z, M, h) | 3D grid keeping BH mass dependence. | |
| Keep 4D (d_L, M, phi, theta) | Same dimensions as current KDE but larger simulation. | |
| **User's choice** | Build full 4D, project down to 2D (z,M) if too noisy | ✓ |

**User's choice:** "Can't we create 4, and then we can always project down onto its 1,2D version?"
**Notes:** Store the full dimensionality, use projected versions for evaluation as needed.

### h-Dependence

| Option | Description | Selected |
|--------|-------------|----------|
| Simulate at true h, remap z↔d_L | Run once, P_det in d_L-space, convert at eval time. | |
| Simulate at multiple h values | Independent campaigns per h value. Most accurate. | |
| You decide | Claude picks. | |

**User's choice:** Initially questioned option 1's research eligibility. After physics discussion about injection campaign methodology (P_det depends on h through population model, mirroring real LISA analysis), agreed on per-h injection campaigns.
**Notes:** Key insight: in real data analysis, P_det is built from simulated injections (not observed events), and detection probability of a source at redshift z depends on the assumed h through d_L(z,h). This mirrors LVK methodology.
