Create `scripts/compare_validation_runs.py` -- a CLI script that takes two run directories and produces a markdown comparison report per D-03, D-04, D-05, D-06, D-10.

Usage: `python scripts/compare_validation_runs.py --baseline evaluation/run_20260328_seed100_v3 --new evaluation/<v12_run_dir> --output evaluation/<v12_run_dir>/comparison_report.md`

Arguments (all required):
- `--baseline`: path to v1.1 run directory
- `--new`: path to v1.2 run directory
- `--output`: path to write the markdown report

The script must:

1. **Load data**: Read `simulations/cramer_rao_bounds.csv` from both directories using pandas. Read `run_metadata_*.json` from both directories for git commit and seed info.

2. **Section 1 -- Run Metadata**: Table with git commit, seed, timestamp, number of tasks for both runs.

3. **Section 2 -- Detection Rate** (per D-03 item 1): Table showing total events (count rows), detections (rows where SNR >= 20), detection rate (detections/total). For both runs side-by-side.

4. **Section 3 -- SNR Distribution** (per D-03 item 2): Summary statistics table (min, median, mean, max, std) of SNR column for detected events. Both runs side-by-side.

5. **Section 4 -- CRB Analysis** (per D-03 item 3):
   - Compute fractional d_L error: `sqrt(abs(delta_luminosity_distance_delta_luminosity_distance)) / luminosity_distance` for each detected event.
   - Summary statistics (min, median, mean, max, percentiles at 10/25/50/75/90/95/99) for both runs.
   - Note: Fisher matrix condition numbers are in SLURM logs, not CSV. Add a note saying "See SLURM logs for condition numbers."

6. **Section 5 -- d_L Threshold Recommendation** (per D-06):
   - Display full percentile table of fractional d_L error from the NEW run only.
   - Recommend a threshold based on the 90th percentile value.
   - Add note per D-07: "Recommendation only -- do NOT update constants.py in this phase."

7. **Section 6 -- Wall Time Analysis** (per D-03 item 4):
   - Compute per-event generation time from `generation_time` column. Summary stats for both runs.
   - Note: Total wall time per task is in SLURM logs, not CSV. Add a note about checking `sacct` output.

8. **Section 7 -- Error Analysis** (per D-10):
   - Count NaN values in SNR column (potential crashes).
   - Count negative CRB diagonal values: check if any `delta_*_delta_*` diagonal entries are negative.
   - Report timeout hit rate and OOB error rate as "See SLURM logs" (these are in stderr, not CSV).

9. **Section 8 -- Pass/Fail Summary** (per D-04):
   - Check 1: "Confusion noise should produce lower SNRs" -- compare median SNR, PASS if v1.2 median < v1.1 median.
   - Check 2: "5-point stencil should produce different CRBs" -- compare median fractional d_L error, PASS if values differ.
   - Check 3: "Detection rate may drop but must remain >0" -- PASS if v1.2 detections > 0.
   - Check 4: "No NaN values" -- PASS if no NaN in SNR column.
   - Check 5: "No negative CRB diagonals" -- PASS if no negative diagonal entries in CRB.

Handle edge cases:
- If v1.2 has 0 detections, report it clearly and mark checks 1,2 as N/A, check 3 as FAIL.
- If CSV files are missing, exit with clear error message.

Use argparse for CLI. Use type annotations on all functions. No matplotlib imports (keep it simple markdown-only).
