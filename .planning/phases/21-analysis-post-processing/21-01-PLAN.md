    Create `master_thesis_code/bayesian_inference/posterior_combination.py` with the following structure:

    1. **CombinationStrategy enum** (per D-06):
       ```python
       class CombinationStrategy(str, Enum):
           NAIVE = "naive"
           EXCLUDE = "exclude"
           PER_EVENT_FLOOR = "per-event-floor"
           PHYSICS_FLOOR = "physics-floor"
       ```

    2. **load_posterior_jsons(posteriors_dir: Path) -> tuple[list[float], dict[int, dict[float, float]]]**:
       - Glob for `h_*.json` files in posteriors_dir
       - Parse each file: extract "h" key, iterate remaining string keys as detection indices
       - Single-element list `[val]` -> store val. Empty list `[]` -> skip (missing event).
       - Return sorted h_values list and event_likelihoods nested dict

    3. **build_likelihood_array(h_values: list[float], event_likelihoods: dict[int, dict[float, float]]) -> tuple[npt.NDArray[np.float64], list[int]]**:
       - Build 2D array [n_events, n_h_values] from the dict structure
       - Events missing an h-value get NaN (distinguish from 0.0 which means "evaluated, got zero")
       - Return the array and sorted list of detection indices

    4. **apply_strategy(likelihoods: npt.NDArray[np.float64], strategy: CombinationStrategy) -> tuple[npt.NDArray[np.float64], int]**:
       - NAIVE: Replace 0.0 with np.finfo(float).tiny, excluded_count=0
       - EXCLUDE: Remove rows (events) that have any 0.0 in any h-bin, return excluded count
       - PER_EVENT_FLOOR: For each event row, replace 0.0 with min(nonzero values in that row) / 100. All-zero events get np.finfo(float).tiny.
       - PHYSICS_FLOOR: Log warning "Physics floor (Option 3) not yet implemented (Phase 22). Falling back to 'exclude' strategy." (per D-07), then delegate to EXCLUDE
       - Return (processed_likelihoods, excluded_count)

    5. **combine_log_space(likelihoods: npt.NDArray[np.float64]) -> npt.NDArray[np.float64]**:
       - `log_likes = np.log(likelihoods)` (input already has zeros replaced by strategy)
       - `joint_log = np.sum(log_likes, axis=0)` — sum across events for each h-bin
       - `max_log = np.max(joint_log)` — shift for numerical stability
       - `posterior = np.exp(joint_log - max_log)`
       - `posterior /= np.sum(posterior)` — normalize
       - Return posterior array [n_h_values]

    6. **generate_diagnostic_report(h_values: list[float], likelihoods: npt.NDArray[np.float64], detection_indices: list[int]) -> str**:
       - Identify events with zeros: which detection indices have 0.0 at which h-bins
       - Categorize: "all-zeros" (zero at every h-bin), "partial-zeros" (zero at some h-bins), "low-h-only" (zeros only at h <= 0.70)
       - Count empty events (NaN rows)
       - Generate markdown report (per D-03, D-04) with sections:
         - `# Zero-Likelihood Diagnostic Report`
         - `## Summary` (total events, zero events, empty events, all-zeros events)
         - `## Zero-Event Detail` (table: Detection Index | Zero h-bins | Pattern)
         - `## Zero Distribution by h-bin` (table: h-value | Number of zeros)
         - `## Root Cause Analysis` (paragraph explaining: all-zeros = no hosts at any h, low-h-only = redshift-dependent catalog gap, partial = coverage boundary)
         - `## Impact on Posterior` (how zeros propagate through naive multiplication)
       - Return the markdown string

    7. **generate_comparison_table(h_values: npt.NDArray[np.float64], likelihoods: npt.NDArray[np.float64], detection_indices: list[int], variant: str) -> str**:
       - Run all 4 strategies, compute MAP for each
       - Generate markdown table (per ANAL-02):
         - `# Posterior Combination Method Comparison`
         - `## Variant: {variant}`
         - Table columns: Strategy | Events Used | Events Excluded | MAP h | MAP Posterior Value
         - Rows for: naive, exclude, per-event-floor, physics-floor
       - Return the markdown string

    8. **combine_posteriors(posteriors_dir: str, strategy: str, output_dir: str) -> dict**:
       - This is the main entry point called from CLI (per D-01, D-02)
       - Parse strategy string to CombinationStrategy enum
       - If strategy is "physics-floor", log warning and fall back to "exclude" (per D-05, D-07)
       - Call load_posterior_jsons, build_likelihood_array, apply_strategy, combine_log_space
       - Generate diagnostic report, write to `{output_dir}/diagnostic_report.md` (per D-03)
       - Generate comparison table, write to `{output_dir}/comparison_table.md`
       - Write `{output_dir}/combined_posterior.json` (per D-08) with schema:
         ```json
         {
           "h_values": [0.6, 0.62, ...],
           "posterior": [0.01, 0.03, ...],
           "strategy": "exclude",
           "n_events_total": 538,
           "n_events_used": 521,
           "n_events_excluded": 17,
           "n_events_empty": 4,
           "map_h": 0.86,
           "map_posterior": 0.15,
           "variant": "posteriors"
         }
         ```
       - Return the dict for programmatic use
