    Present the full /physics-change review for PE-01 before writing any code. The user must
    approve before proceeding to Task 2.

    **Old formula (parameter_space.py ~line 148):**
      self.luminosity_distance.value = dist(host_galaxy.z)
    Uses the hardcoded H=0.73 constant (default argument in dist() from physical_relations.py).
    When h_inj = 0.73 (standard campaign), this is harmless. When h_inj ≠ 0.73 (injection
    sweep campaigns), the Fisher CRBs are computed at the wrong luminosity distance.

    **New formula:**
      self.luminosity_distance.value = dist(host_galaxy.z, h=h)
    where h: float is a required argument passed from the simulation caller (h_value at main.py:394).

    **Reference:**
    Dark siren PE standard practice: Fisher CRBs must be computed self-consistently at the
    injected Hubble constant h_inj (Gray et al. 2020, Laghi et al. 2021, Finke et al. 2021).
    The dist() function in physical_relations.py already accepts h as a keyword argument — this
    fix threads the existing h parameter into the existing function call.

    **Dimensional analysis:**
    - h: dimensionless (ratio H0 / [100 km/s/Mpc])
    - dist(z, h=h): returns luminosity distance in Mpc (same units as before)
    - luminosity_distance.value: float, in Mpc
    - No dimensional change; the formula structure is unchanged; only the value of h differs.

    **Limiting case:**
    When h = H = 0.73 (the hardcoded default), dist(host_galaxy.z, h=0.73) = dist(host_galaxy.z).
    The new formula produces identically the same result as the old formula for the standard campaign.
    The change only has an effect when h_inj ≠ 0.73, which is the intended fix.

    **Sign convention:** luminosity_distance is positive-definite; dist() returns d_L > 0 for z > 0.
    No sign change.
