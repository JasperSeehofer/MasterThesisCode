Implement the condition-number gate for Fisher matrix quality in `bayesian_statistics.py`, add the `--fisher_cond_threshold` CLI flag, write `fisher_quality.csv` diagnostic output, and run an empirical calibration to set the default threshold.

Purpose: Near-singular covariance matrices currently pass through `pinv()` silently, potentially corrupting posterior likelihoods. This plan adds explicit detection and exclusion of degenerate events with full diagnostic logging.

Output: Working condition-number gate with CSV diagnostics, CLI flag, unit tests, and empirically determined default threshold.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CSV input -> covariance construction | CRB CSV values become covariance matrices; extreme values could cause numerical overflow in cond() or eigh() |
| CLI threshold -> evaluation logic | User-supplied threshold controls which events are excluded; extreme values (0 or inf) change behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-01 | T (Tampering) | fisher_quality.csv | accept | Diagnostic output only; not a security boundary. Written to local filesystem alongside existing posteriors. |
| T-34-02 | D (Denial of Service) | np.linalg.cond() on degenerate matrix | accept | cond() on a 3x3/4x4 matrix is O(1) even for singular inputs; cannot cause hang or OOM. |
| T-34-03 | I (Information Disclosure) | Condition numbers in log output | accept | No sensitive data; condition numbers are diagnostic metrics for numerical quality. |
| T-34-04 | T (Tampering) | --fisher_cond_threshold CLI | accept | Local CLI argument; no network attack surface. Unreasonable values (<=0) would exclude all or no events, which is safe (evaluation still produces valid output). |
