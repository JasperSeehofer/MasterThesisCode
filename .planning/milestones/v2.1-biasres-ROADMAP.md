# Milestone v2.1: H₀ Bias Resolution

**Status:** ✅ SHIPPED 2026-04-09
**Phases:** 30–34
**Total Plans:** 10 plans (9 GSD + 1 GPD)

## Overview

Systematic diagnosis and elimination of the H₀ posterior bias that was causing the MAP to peak at h=0.66 instead of the injected h=0.73. Root cause identified as an incorrect L_comp denominator (local window instead of full detectable volume). All five planned phases executed; bias eliminated (0.0% at N=59, ~1.4% residual at full 417-event dataset consistent with statistical scatter).

### Phase 31: Catalog-Only Diagnostic

**Goal**: Confirm L_comp is the dominant bias source by running with f_i=1.0
**Depends on**: Phase 30
**Plans**: 2 plans

Plans:
- [x] 31-01-PLAN.md — --catalog_only CLI flag, completion term bypass, per-event diagnostic CSV
- [x] 31-02-PLAN.md — Diagnostic summary generator, comparison flow integration, end-to-end verification

**Details:**
`--catalog_only` flag bypasses L_comp (sets f_i=1.0 for all events). Per-event CSV logs L_cat, L_comp, f_i, combined log-likelihood at each h. Diagnostic summary generator computes L_comp weight fractions and pull-direction statistics. Catalog-only MAP confirmed to shift toward h=0.73, validating L_comp as dominant bias source.

### Phase 32: Completion Term Fix (GPD)

**Goal**: Replace incorrect L_comp denominator with full-volume D(h) integral
**Depends on**: Phases 30, 31
**Plans**: 2 plans

Plans:
- [x] 32-01-PLAN.md — precompute_completion_denominator(), D_h_table, zero-fill P_det accessor
- [x] 32-02-PLAN.md — Validation: MAP comparison, bias-vs-N convergence, L_comp decomposition

**Details:**
Root cause: L_comp denominator was integrated over the local 4-sigma d_L window instead of the full detectable volume. Fix: `precompute_completion_denominator()` integrates P_det * dVc/dz over [1e-6, z_max(h)] once per h-value. Per Gray et al. (2020) Eq. A.19. Result: MAP 0.60→0.73 (true value), bias -17.8%→0.0% at N=59 events. Full artifacts in `.gpd/phases/32-completion-term-fix/`.

### Phase 33: P_det Grid Resolution

**Goal**: Validate that P_det grid resolution (30→60 bins) does not affect the posterior
**Depends on**: Phase 30
**Plans**: 2 plans

Plans:
- [x] 33-01-PLAN.md — Configurable P_det bins via CLI (--pdet_dl_bins, --pdet_mass_bins), coverage validation, tests
- [x] 33-02-PLAN.md — Cluster script update + full 38-point h-sweep + before/after comparison

**Details:**
Made P_det grid resolution configurable. Increased default from 30 to 60 d_L bins, 20 to 40 mass bins. Full cluster sweep (38 h-values, jobs 3897259 + retry 3898710 + combine 3898840) confirmed **zero delta** vs 30-bin baseline — log-posteriors identical to 4 decimal places. P_det grid resolution ruled out as bias source.

### Phase 34: Fisher Matrix Quality

## Milestone Summary

**Key Decisions:**
- Phase 32 used full-volume D(h) denominator per Gray et al. (2020) Eq. A.19 rather than a rate-model prior (simpler, physically motivated, matches the reference implementation)
- Phase 33 validated via full cluster sweep rather than local single-h test (user request — more definitive)
- Phase 34 uses exclusion (return [0.0]) rather than regularization for singular matrices — conservative choice, flagged in diagnostics

**Issues Resolved:**
- H₀ posterior bias: MAP 0.60→0.73 (dominant bias source eliminated by Phase 32)
- P_det grid resolution confirmed not a bias source (Phase 33)
- allow_singular=True removed; Fisher quality monitoring established (Phase 34)

**Issues Deferred (not blocking):**
- 1.4% residual bias at N=417 — consistent with statistical scatter; Phase 32 shows 0.0% at N=59
- wCDM params w0, wa silently ignored (PHYS-01)
- Pipeline A hardcoded 10% σ(d_L) (PHYS-02)
- WMAP-era cosmology constants (PHYS-03)
- Galaxy redshift uncertainty (1+z)^3 scaling (PHYS-04)

**Technical Debt Incurred:**
- None significant. All fixes properly tested and documented.
