  <files>N/A (human web portal and terminal actions)</files>
  <action>
    User must complete two manual steps that Claude cannot automate:

    **Step 1: Register SSH key on bwIDM portal (ACCESS-01)**
    1. Open https://login.bwidm.de/user/ssh-keys.xhtml in a browser
    2. Log in with university credentials
    3. Click "Add SSH Key" or equivalent
    4. Paste the contents of `~/.ssh/id_ed25519.pub` (run `cat ~/.ssh/id_ed25519.pub` to get it)
    5. Register the key as **Interactive** for the **bwUniCluster 3.0** service
    6. Confirm the key shows as "Active" in the portal
    NOTE: There may be a propagation delay (minutes to hours) before the key is usable.

    **Step 2: Perform initial 2FA login (ACCESS-02)**
    1. Ensure VPN is connected (if required)
    2. Run: `ssh bwunicluster`
    3. Enter TOTP code and service password when prompted
    4. Verify shell prompt appears on the cluster
    5. Type `exit` (ControlMaster keeps the connection alive in background)

    **Step 3: Verify ControlMaster**
    1. Run: `ssh bwunicluster 'hostname'`
    2. Should return hostname immediately (no 2FA prompt)
    3. Expected output: something like `uc3-login1` or similar

    Troubleshooting: If "Permission denied", wait for key propagation and retry.
    If hostname does not resolve, try changing HostName to `uc3.scc.kit.edu` in ~/.ssh/config.
  </action>
