2. Create `master_thesis_code/plotting/interactive.py` with these 4 factory functions.
   All functions return `plotly.graph_objects.Figure` and take the same data types as
   their matplotlib counterparts. Use `include_plotlyjs="cdn"` in all `.write_html()` calls
   to minimize file sizes.

   **Color palette:** Import from `_colors.py` and reuse the Okabe-Ito hex strings directly
   (CYCLE, TRUTH, MEAN, EDGE, REFERENCE, ACCENT). Plotly accepts hex colors natively.

   **Label mapping:** Import LABELS from `_labels.py`. Strip LaTeX markup for Plotly axis labels
   using a small helper `_strip_latex(label: str) -> str` that converts e.g.
   `r"$M_\bullet \, [M_\odot]$"` to `"M_bullet [M_sun]"` (simple regex replacements for
   `\bullet` -> `bullet`, `\odot` -> `sun`, strip `$`, `\,`, `\mathrm{...}` -> contents).
   This is NOT a general LaTeX parser -- just handle the specific patterns in LABELS.

   **Function signatures:**

   a) `interactive_combined_posterior(h_values, posterior, true_h, *, label=None, show_credible=True, show_references=True) -> go.Figure`:
      - Main posterior as filled line trace
      - 68% and 95% credible interval shading (vertical span shapes)
      - Planck and SH0ES reference bands as vertical rect shapes with annotations
      - Truth vertical line
      - Hover shows h value and posterior density
      - Legend toggle for each element

   b) `interactive_sky_map(theta_s, phi_s, snr, *, redshifts=None, distances=None) -> go.Figure`:
      - Scattergeo with Mollweide-like layout (use `go.Scattergeo` with `projection_type="mollweide"`)
      - Color by SNR using viridis colorscale
      - Hover template shows: event index, SNR, and optionally redshift + distance if provided
      - Coordinate transform: colatitude theta_s [0,pi] -> lat [-90,90], phi_s [0,2pi] -> lon [-180,180]

   c) `interactive_fisher_ellipses(events, pairs=None, *, sigma_levels=(1.0, 2.0)) -> go.Figure`:
      - `events` is `list[tuple[NDArray, NDArray]]` -- list of (14x14 covariance, 14-element param_values)
      - Default pairs: same as `fisher_plots._DEFAULT_PAIRS` = [("M","mu"), ("luminosity_distance","qS"), ("qS","phiS")]
      - Use subplots (one column per pair) via `plotly.subplots.make_subplots`
      - For each event, for each pair, draw ellipse as a parametric curve (theta 0..2pi, transform by eigenvectors)
        using `go.Scatter` with mode="lines", filled
      - Hover on ellipse shows event index and sigma level
      - Zoom/pan enabled by default
      - Import `_ellipse_params` from `fisher_plots` for eigendecomposition, but generate the parametric
        ellipse points yourself (100-point polygon from the eigenvalues/eigenvectors)
      - Use CYCLE colors per event, alpha via opacity

   d) `interactive_h0_convergence(h_values, event_posteriors, *, true_h=None, subset_sizes=None, seed=42, level=0.68) -> go.Figure`:
      - Two-panel layout via `make_subplots(rows=1, cols=2)`
      - Left: combined posterior curves for increasing N (same logic as `convergence_plots.plot_h0_convergence`)
      - Right: CI width vs N with 1/sqrt(N) reference
      - Hover on left panel shows h and density; hover on right shows N and CI width
      - Truth line on left panel if provided

   Also add a top-level convenience function:
   `generate_all_interactive(output_dir, data_dir) -> list[str]`:
      - Loads data from `data_dir` (CRB CSVs via `_load_crb_data()` helper, posterior JSONs via
        `load_posterior_jsons` from `bayesian_inference.posterior_combination`)
      - Calls each of the 4 functions with loaded data
      - Writes HTML files to `output_dir`: `combined_posterior.html`, `sky_map.html`,
        `fisher_ellipses.html`, `h0_convergence.html`
      - Uses `fig.write_html(path, include_plotlyjs="cdn")`
      - Returns list of written file paths
      - Gracefully skips figures when data is missing (log warning, continue)
      - Uses a `_load_crb_data(data_dir)` internal helper similar to `main.py:generate_figures`
      - For posteriors, tries both `posteriors/` and `posteriors_with_bh_mass/` subdirs

   **Type annotations:** Full mypy-strict compliance. All parameters and returns annotated.
   Use `npt.NDArray[np.float64]` for arrays. Import `plotly.graph_objects as go`.
